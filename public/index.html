⚡ REST API

Pokedex API

API REST construida con NestJS y MongoDB que permite gestionar un catálogo completo de Pokémon. Soporta creación, consulta, actualización y eliminación, con paginación y búsqueda flexible por nombre, número o ID.

NestJS v10 MongoDB + Mongoose TypeScript Docker ready REST

Tech Stack

🐱
NestJS Framework principal
🍃
MongoDB Base de datos NoSQL
🔷
TypeScript Tipado estático
🐋
Docker Contenedores
🪶
Mongoose ODM para MongoDB
✅
class-validator Validación de DTOs

Instalación

Clonar el repositorio

git clone https://github.com/tu-usuario/pokedex.git
cd pokedex

Instalar dependencias

npm install

Variables de entorno

Crea un archivo .env en la raíz del proyecto:

MONGODB=mongodb://localhost:27017/nest-pokemon
PORT=3000
DEFAULT_LIMIT=10

Levantar MongoDB con Docker

docker compose up -d

Correr en modo desarrollo

npm run start:dev

La API estará disponible en http://localhost:3000

Poblar la base de datos

Ejecuta el seed para cargar los primeros 650 Pokémon desde la PokeAPI:

GET http://localhost:3000/api/seed

Endpoints

Seed

GET /api/seed Pobla la BD con datos de PokeAPI
Response 200
"Seed executed"
GET /api/pokemon Lista Pokémon con paginación
Query params
Campo Tipo Default
limit number 10 opcional
offset number 0 opcional
Response 200
[
  {
    "_id": "664f1a...",
    "no":   1,
    "name": "bulbasaur"
  },
  ...
]
GET /api/pokemon/:term Busca por nombre, número o MongoID
Búsqueda flexible — el parámetro :term puede ser el nombre (pikachu), el número de Pokédex (25) o el _id de MongoDB.
Response 200
{
  "_id":  "664f1a...",
  "no":   25,
  "name": "pikachu"
}
POST /api/pokemon Crea un nuevo Pokémon
Body (JSON)
{
  "no":   152,
  "name": "chikorita"
}
Response 201
{
  "_id":  "664f2b...",
  "no":   152,
  "name": "chikorita"
}
PATCH /api/pokemon/:term Actualiza nombre o número de un Pokémon
Body (JSON) — todos los campos son opcionales
{
  "name": "chikorita-updated"
}
DELETE /api/pokemon/:id Elimina un Pokémon por MongoID

Retorna 204 No Content si fue eliminado. El parámetro :id debe ser un ObjectId válido de MongoDB.

Schema del Pokémon

Campo Tipo Restricciones
_id ObjectId Auto-generado por MongoDB auto
no number Único, positivo, mínimo 1 requerido
name string Único, mínimo 1 carácter, lowercase requerido

Respuestas de error

400 Bad Request Validación fallida o Pokémon duplicado
404 Not Found Pokémon no encontrado por el término dado
500 Internal Server Error Error inesperado del servidor
